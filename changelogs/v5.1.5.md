## v5.1.5 - Agent 核心文件、网关探测增强与飞书切换修复

### 重点更新
- **feat**: `Agents` 页面补齐官方风格工作区能力
  - 新增 `核心文件 (Core Files)` 查看与保存
  - 新增 `技能与上下文 (Skills · Channels · Cron)` 快照
  - 新增 `最近会话 (Recent Sessions)` 与更完整的 `Tools & Access` 摘要
- **feat**: 后端新增 Agent 核心文件接口
  - `GET /api/openclaw/agents/:id/core-files`
  - `PUT /api/openclaw/agents/:id/core-files`
  - 仅允许访问受管工作区下的核心文件白名单

### 稳定性修复
- **fix**: ClawPanel 启动时能更准确识别已由外部管理的 OpenClaw 网关，避免重复拉起
- **fix**: 网关端口探测按 `gateway.bind` 绑定模式选择目标地址，减少 loopback / LAN / tailnet 误判
- **fix**: Windows `go vet` 的 `unsafe.Pointer` 问题已修复，`windows-2025` CI 恢复通过
- **fix**: `CreateEnvironmentBlock` 失败时，NapCat 启动会回退到当前进程环境基线，避免缺失关键环境变量
- **fix**: 飞书双版本切换现在会检查目标插件是否已安装，避免切换到不存在的实现
- **fix**: 飞书通道页未安装的版本会禁用切换入口并给出提示

- 已通过：`GOOS=windows GOARCH=amd64 go vet ./internal/monitor/...`
- 已通过：`cd web && npm run build`
