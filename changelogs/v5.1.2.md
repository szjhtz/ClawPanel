## v5.1.2 - Agents 弹窗结构化配置与 implicit main 语义修复

### 重点改进
- **feat**: `Agents` 新建/编辑弹窗升级为结构化配置器
  - 分为 `Basic / Behavior / Access & Safety / Collaboration / Advanced`
  - 首屏改为任务导向表单，不再默认暴露大段 JSON
  - 增加默认模型提示、快捷模型 chips、默认 Agent 行为说明
- **feat**: 弹窗 header / footer 改为 sticky 布局
  - 1366×768 笔记本视口下，保存按钮保持可见
  - 保存前给出更明确的阻塞原因与引导文案

### 语义修复
- **fix**: implicit `main` 占位 Agent 现在有明确语义
  - 后端 `GET /api/openclaw/agents` 会为运行时合成的占位项返回 `implicit=true`
  - 前端据此显示 `PLACEHOLDER`、使用“配置”而不是“编辑”，并禁用删除
  - 当用户首次保存 `main` 时，会正确物化为显式 Agent；保存后自动切回普通“编辑 / 删除”状态
  - 当 `agents.list` 缺失但磁盘上已发现 agent 目录时，这些发现项也会按 implicit Agent 处理，避免直接 update/delete 命中 404
  - legacy single-agent 模式显式返回 `implicit=false`，不会被误判成 placeholder
- **fix**: Advanced JSON 不再因结构化保存而静默丢失未知字段
  - 仅在用户真正改动对应结构化分组时，才覆盖 `model / params / identity / groupChat / tools / subagents`
  - 未触碰的高级 JSON 会原样保留

### 测试

  - `TestGetOpenClawAgentsKeepsExplicitMainNonImplicit`
- 已通过：`go test ./...`
- 已通过：`npm --prefix web run build`

### 影响说明
- 前端现在会把“运行时占位 Agent”和“显式配置 Agent”区分显示，避免用户误以为占位 `main` 已经写入 `agents.list`。
- 如果你仅修改 Basic 信息而不触碰 Advanced 对应结构化分组，原有高级字段会继续保留，不需要重新手写 JSON。
